# Step 0.2: Problem Statement

## 핵심 문제 한 줄 정의
> 사람들은 AI 에이전트를 "팀"처럼 조합해서 활용할 수 있다는 가능성 자체를 인식하지 못하며, 안다고 해도 설정이 어렵고 과정이 보이지 않아 제대로 활용하지 못한다.

## 문제 구조 (3단계)

### Level 1: 인식의 부재 (Awareness Gap)
- **문제**: AI 에이전트를 팀처럼 조합해서 쓸 수 있다는 생각 자체를 못 함
- **원인**: 기존 AI 도구들이 "하나의 채팅창"이라는 단일 인터페이스만 제공
- **결과**: AI의 잠재력을 극히 일부만 활용

### Level 2: 진입 장벽 (Setup Barrier)
- **문제**: AI 에이전트를 설정하고 구성하는 것이 너무 어려움
- **원인**: 기술적 지식(프롬프트 엔지니어링, API 설정 등)이 필요
- **결과**: 비개발자는 시도조차 못 함

### Level 3: 경험의 불투명성 (Black Box Problem)
- **문제**: AI에게 일을 시키고 결과를 받는 과정이 직관적이지 않음
- **원인**: 중간 과정이 보이지 않고, 결과만 텍스트로 반환됨
- **결과**: 신뢰 부족, 통제감 상실, 이탈

## 타겟 사용자의 현재 상태
- AI를 써보고 싶지만 어디서 시작해야 할지 모르는 비개발자
- ChatGPT 정도는 써봤지만, 에이전트/자동화까지는 가지 못한 사람
- "AI가 뭘 할 수 있는지"를 구체적으로 모르는 사람

## 이 아이디어의 해결 전략
> **"회사에 팀을 만드는 것"이라는 익숙한 메타포** + **Gather 스타일 게임 비주얼**

| 문제 | 해결 방식 |
|------|-----------|
| 인식의 부재 | "팀을 만든다"는 현실 세계의 메타포로 가능성을 자연스럽게 전달 |
| 진입 장벽 | 드래그 앤 드롭 + 자연어 설명으로 코딩/설정 없이 팀 구성 |
| 블랙박스 | 가상 오피스에서 에이전트가 일하는 모습을 실시간 시각화 |
