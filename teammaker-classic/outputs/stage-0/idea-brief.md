# TeamMaker - Idea Brief

## 제품 한 줄 설명
> Gather 스타일 가상 오피스에서 책상을 배치해 AI 에이전트 팀을 만들고, 자연어로 일을 시키면 어떤 팀이 처리 중인지 실시간으로 보여주는 서비스

---

## 1. 문제 정의

### 핵심 문제
사람들은 AI 에이전트를 "팀"처럼 조합해서 활용할 수 있다는 **가능성 자체를 인식하지 못하며**, 안다고 해도 설정이 어렵고 과정이 보이지 않아 제대로 활용하지 못한다.

### 문제 구조

| 레벨 | 문제 | 현재 상태 |
|------|------|-----------|
| **인식** | AI를 팀으로 쓸 수 있다는 생각 자체를 못 함 | 1:1 채팅만 알고 있음 |
| **진입** | 에이전트 설정/구성이 너무 어려움 | 비개발자는 시도 불가 |
| **경험** | AI가 뭘 하고 있는지 보이지 않음 | 블랙박스, 신뢰 부족 |

### 타겟 사용자
- 비개발자 및 AI 이해도가 낮은 사람
- ChatGPT는 써봤지만 에이전트/자동화까지는 못 간 사람
- "AI가 뭘 할 수 있는지"를 구체적으로 모르는 사람

---

## 2. 솔루션

### 해결 전략
**"회사에 팀을 만드는 것"이라는 익숙한 메타포** + **Gather 스타일 게임 비주얼**

| 문제 | 해결 방식 |
|------|-----------|
| 인식의 부재 | "팀을 만든다"는 현실 메타포로 가능성을 자연스럽게 전달 |
| 진입 장벽 | 드래그 앤 드롭 + 자연어 설명만으로 팀 구성 |
| 블랙박스 | 가상 오피스에서 에이전트 작업을 실시간 시각화 |

### MVP 핵심 기능 3가지

#### 1. 가상 오피스 캔버스
- 2D 탑뷰 사무실 공간 (Gather 스타일)
- 빈 사무실에서 시작
- 그리드 기반 공간에 팀 데스크를 자유롭게 배치
- 상태에 따라 시각적 변화 (대기 → 작업 중 → 완료)

#### 2. 팀 생성
1. 팔레트에서 빈 책상을 드래그 → 오피스에 드롭
2. 팀 이름 + 설명 입력 (예: "고객응대팀: 고객 문의에 답변")
3. AI가 설명을 이해하여 에이전트 구성 자동 제안
4. 사용자 확인/수정 후 완료
- 유연한 구조: 단일 에이전트 또는 복수 에이전트 묶음

#### 3. 업무 지시 & 실시간 시각화
- 채팅으로 자연어 지시 → AI가 적절한 팀 자동 매칭
- **팀 활성 상태**: 작업 중인 데스크가 시각적으로 활성화
- **팀 간 협업 흐름**: 팀A → 팀B로 결과가 이동하는 과정 시각화
- 클릭하면 상세 진행 상황 확인

## 3. 사용자 여정

```
[빈 사무실]
"나만의 AI 회사를 만들어보세요!"

     ↓ 책상 드래그 & 드롭

[팀 생성]
"고객응대팀: 고객 문의에 친절하게 답변하는 팀"
→ AI 제안: "메일 응답 에이전트 + FAQ 검색 에이전트"
→ 확인!

     ↓ 채팅으로 업무 지시

[업무 진행]
"어제 들어온 고객 문의 3건 답변 초안 만들어줘"
→ 고객응대팀 데스크 활성화 (빛남!)
→ FAQ 에이전트 작동 → 메일 에이전트로 결과 전달
→ 팀 간 데이터 이동 애니메이션

     ↓ 완료

[결과 확인]
채팅에 결과물 표시 + 데스크에 완료 표시
데스크 클릭 → 작업 히스토리 & 결과물 확인
```

## 5. 기술 아키텍처

### 핵심 기술 선택

| 레이어 | 기술 | 근거 |
|--------|------|------|
| 캔버스 렌더링 | **PixiJS v8** | Gather.town이 사용하는 동일 기술. 60fps에서 1000+ 스프라이트 처리 |
| React 연동 | **@pixi/react v8** | JSX로 선언적 캔버스 작성 |
| 애니메이션 | **GSAP** | Gather.town과 동일. 상태 전환에 최적 |
| 실시간 통신 | **Socket.io** | 재연결, 룸 기능 기본 지원 |
| 프레임워크 | **React + TypeScript** | @pixi/react와 자연스럽게 연결 |
| 상태 관리 | **Zustand** | 경량, 실시간 상태에 적합 |
| AI | **Anthropic Claude API** | 사용자 API Key 입력 방식 |

### 아키텍처 다이어그램

```
┌─────────────────────────────────────────┐
│             프론트엔드 (React)            │
│  ┌──────────────┐  ┌─────────────────┐  │
│  │  PixiJS 캔버스 │  │  UI (채팅, 패널)  │  │
│  │  (가상 오피스)  │  │  (React 컴포넌트) │  │
│  └──────┬───────┘  └────────┬────────┘  │
│         └────── Zustand ─────┘           │
└──────────────┬──────────────────────────┘
               │ WebSocket + REST
┌──────────────┴──────────────────────────┐
│               백엔드 (Node.js)            │
│  ┌──────────────┐  ┌─────────────────┐  │
│  │  WebSocket    │  │  에이전트        │  │
│  │  서버         │  │  오케스트레이터   │  │
│  └──────────────┘  └────────┬────────┘  │
│                              │           │
│                     Claude API (Tool Use) │
└──────────────────────────────────────────┘
```

### 오픈소스 레퍼런스
- [gather-clone](https://github.com/trevorwrightdev/gather-clone): Next.js + Pixi.js + Socket.io - **가장 유사한 구조**
- [WorkAdventure](https://github.com/workadventure/workadventure): 가장 성숙한 오픈소스 Gather 대안

## 6. 기술 타당성

### 종합 평가: 8/10

| 항목 | 점수 | 이유 |
|------|------|------|
| 2D 캔버스 구현 | 9/10 | PixiJS는 검증된 기술, Gather가 실제 사용 |
| 드래그앤드롭 | 8/10 | PixiJS 포인터 이벤트 + 그리드 스냅으로 구현 |
| 실시간 시각화 | 8/10 | WebSocket + GSAP 조합으로 충분 |
| AI 팀 자동 구성 | 7/10 | Claude Tool Use로 가능하나 프롬프트 설계 필요 |
| 에이전트 간 협업 | 7/10 | 파이프라인 방식 구현 가능, 복잡도 관리 필요 |

### MVP 일정 (1-2명, 약 8-10주)

| 마일스톤 | 기간 |
|----------|------|
| 오피스 캔버스 + 책상 배치 | 2-3주 |
| 팀 생성 플로우 (AI 포함) | 2주 |
| 업무 지시 + 실시간 시각화 | 2-3주 |
| 폴리싱 + 테스팅 | 2주 |
