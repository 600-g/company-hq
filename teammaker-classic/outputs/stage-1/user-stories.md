# Step 1.2: User Stories

## 페르소나

**민지** (29세, 스타트업 마케팅 담당)
- AI 도구는 ChatGPT 정도만 써봄
- "AI가 더 많은 걸 할 수 있다는데 뭘 어떻게 해야 할지 모르겠다"
- 코딩 경험 없음, 기술 용어에 거부감
- 반복 업무가 많아서 AI에 위임하고 싶지만 방법을 모름

---

## Epic 1: 가상 오피스 (Sprint 1)

### US-1.1: 오피스 둘러보기
> 사용자로서, 서비스에 접속하면 빈 사무실 공간이 보이고, 여기에 내 AI 팀을 채울 수 있다는 걸 직관적으로 알 수 있어야 한다.

**인수 조건**:
- [ ] 접속 시 2D 탑뷰 오피스 공간이 화면에 표시된다
- [ ] 오피스는 그리드 기반으로 정렬된 빈 공간이다
- [ ] "여기에 팀을 배치해보세요" 같은 안내 메시지가 보인다
- [ ] 마우스 드래그로 캔버스를 이동(패닝)할 수 있다
- [ ] 스크롤/핀치로 줌인/줌아웃할 수 있다

### US-1.2: 오피스 커스터마이징 (기본)
> 사용자로서, 오피스 공간의 전체 모습을 보면서 어디에 어떤 팀을 놓을지 자유롭게 정할 수 있어야 한다.

**인수 조건**:
- [ ] 배치된 데스크의 위치를 드래그로 변경할 수 있다
- [ ] 데스크가 그리드에 스냅되어 깔끔하게 정렬된다

---

## Epic 2: 팀 생성 (Sprint 1)

### US-2.1: 책상 배치하기
> 사용자로서, 화면 한쪽에 있는 빈 책상을 드래그해서 오피스 원하는 위치에 놓을 수 있어야 한다. 마치 게임에서 가구를 배치하는 것처럼.

**인수 조건**:
- [ ] 화면 사이드에 "빈 책상" 아이템이 있는 팔레트가 보인다
- [ ] 빈 책상을 드래그하면 커서를 따라 이동한다
- [ ] 오피스 그리드 위에 드롭하면 해당 위치에 책상이 놓인다
- [ ] 드롭 즉시 팀 정보 입력 폼이 열린다

### US-2.2: 팀 설명하기
> 사용자로서, "이 팀이 뭘 하는 팀인지"를 내 말로 자유롭게 설명하면 AI가 알아서 이해해야 한다. 기술 용어를 쓸 필요 없이.

**인수 조건**:
- [ ] 팀 이름을 입력할 수 있다 (예: "고객응대팀")
- [ ] 팀 설명을 자유롭게 입력할 수 있다 (예: "고객 문의 메일에 친절하게 답변해주는 팀")
- [ ] 입력 폼에 예시가 플레이스홀더로 제공된다
- [ ] "팀 만들기" 버튼을 누르면 AI가 분석을 시작한다

### US-2.3: AI 팀 구성 확인하기
> 사용자로서, AI가 내 설명을 이해해서 제안한 에이전트 구성을 보고, 이해할 수 있어야 한다. "아, 이런 역할들로 팀이 구성되는구나" 하고.

**인수 조건**:
- [ ] AI가 팀 설명을 분석하여 에이전트 구성을 제안한다
- [ ] 각 에이전트의 역할이 쉬운 말로 설명된다 (예: "메일 작성 담당", "자료 검색 담당")
- [ ] 로딩 중에는 "팀을 구성하고 있어요..." 같은 피드백이 보인다
- [ ] 제안된 구성을 "확인" 버튼으로 수락할 수 있다

### US-2.4: 팀 구성 수정하기
> 사용자로서, AI가 제안한 팀 구성이 맘에 안 들면 수정할 수 있어야 한다.

**인수 조건**:
- [ ] 제안된 에이전트를 삭제할 수 있다
- [ ] 에이전트의 역할 설명을 수정할 수 있다
- [ ] "에이전트 추가" 버튼으로 새 에이전트를 추가할 수 있다
- [ ] 수정 후 "확인"하면 팀이 최종 생성된다
- [ ] 생성된 팀이 데스크 위에 시각적으로 표현된다

---

## Epic 3: 업무 지시 (Sprint 2)

### US-3.1: 일 시키기
> 사용자로서, 채팅창에 하고 싶은 말을 쓰면 AI 팀이 알아서 처리를 시작해야 한다. "이 팀에게 이 일을 시켜라" 같은 복잡한 명령 없이.

**인수 조건**:
- [ ] 화면 하단 또는 사이드에 채팅 입력창이 있다
- [ ] 자연어로 업무를 입력할 수 있다 (예: "블로그 글 3개 써줘")
- [ ] AI가 적절한 팀을 자동으로 매칭한다
- [ ] 어떤 팀에게 일이 할당되었는지 채팅에 표시된다
- [ ] 특정 팀을 지정해서 일을 시킬 수도 있다 (예: "@마케팅팀 블로그 글 써줘")

### US-3.2: 진행 상황 눈으로 보기
> 사용자로서, 일을 시키고 나면 오피스에서 어떤 팀이 일하고 있는지 바로 눈에 보여야 한다. 검은 화면에서 텍스트가 쭉 올라오는 게 아니라.

**인수 조건**:
- [ ] 작업 중인 팀의 데스크가 시각적으로 활성화된다 (빛남, 애니메이션 등)
- [ ] 대기 중인 팀과 작업 중인 팀이 시각적으로 명확히 구분된다
- [ ] 작업이 완료되면 데스크 상태가 "완료"로 변한다

### US-3.3: 상세 진행 상황 확인
> 사용자로서, 일하고 있는 팀의 데스크를 클릭하면 지금 구체적으로 뭘 하고 있는지 볼 수 있어야 한다.

**인수 조건**:
- [ ] 데스크 클릭 시 사이드 패널이 열린다
- [ ] 패널에 현재 진행 중인 작업 설명이 표시된다
- [ ] 팀 내 각 에이전트가 무슨 일을 하고 있는지 보인다
- [ ] 완료된 작업의 결과물을 확인할 수 있다

---

## Epic 4: 팀 협업 시각화 (Sprint 3)

### US-4.1: 팀 간 일 넘기기 보기
> 사용자로서, 한 팀이 다른 팀에게 결과를 넘기는 과정이 오피스에서 시각적으로 보여야 한다. 마치 직원이 서류를 들고 다른 팀 책상으로 걸어가는 것처럼.

**인수 조건**:
- [ ] 팀A가 작업을 완료하고 팀B에게 넘기면, 오피스 위에서 이동 애니메이션이 보인다
- [ ] 어떤 데이터/결과물이 이동하는지 간단한 라벨로 표시된다
- [ ] 받는 팀(B)이 자동으로 활성화되어 작업을 시작한다
- [ ] 전체 협업 흐름이 순서대로 시각적으로 진행된다

### US-4.2: 복잡한 업무 위임하기
> 사용자로서, "신제품 런칭 캠페인 만들어줘" 같은 큰 업무를 시키면 여러 팀이 순서대로 또는 동시에 협업하여 처리하는 걸 볼 수 있어야 한다.

**인수 조건**:
- [ ] 큰 업무를 시키면 AI가 하위 작업으로 분해한다
- [ ] 각 하위 작업이 적절한 팀에 할당된다
- [ ] 팀 간 의존 관계가 있으면 순서대로, 독립적이면 동시에 처리된다
- [ ] 전체 진행률을 한눈에 파악할 수 있다

---

## Epic 5: 기본 설정 (Sprint 1)

### US-5.1: API Key 입력
> 사용자로서, 서비스를 시작할 때 Anthropic API Key를 입력하면 바로 사용할 수 있어야 한다.

**인수 조건**:
- [ ] 최초 접속 시 API Key 입력 화면이 표시된다
- [ ] API Key를 입력하고 "시작" 버튼을 누르면 검증된다
- [ ] 유효한 Key면 오피스 화면으로 이동한다
- [ ] Key는 로컬에 안전하게 저장되어 재방문 시 다시 입력 안 해도 된다

---

## Epic 6: 온보딩 (Sprint 3)

### US-6.1: 첫 팀 만들기 가이드
> 사용자로서, 처음 빈 사무실을 봤을 때 뭘 해야 하는지 바로 알 수 있어야 한다. 복잡한 튜토리얼 없이.

**인수 조건**:
- [ ] 빈 사무실에 "책상을 드래그해서 첫 팀을 만들어보세요!" 안내가 보인다
- [ ] 팔레트의 빈 책상에 하이라이트/깜빡임으로 시선 유도
- [ ] 첫 팀 생성 완료 시 축하 피드백 (간단한 이펙트)
- [ ] 이후 "이제 채팅에서 일을 시켜보세요!" 안내 표시
