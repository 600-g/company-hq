# Step 1.3: User Flow

## 전체 흐름 개요

```
[접속] → [API Key 입력] → [빈 오피스] → [팀 생성] → [업무 지시] → [실시간 관찰] → [결과 확인]
```

---

## Flow 1: 첫 접속 → 첫 팀 생성 (핵심 흐름)

이 흐름이 매끄러워야 사용자가 이탈하지 않습니다.

```
┌─────────────────────────────────────────────────────────────┐
│                    FLOW 1: 첫 팀 만들기                       │
└─────────────────────────────────────────────────────────────┘

[1. 랜딩/접속]
    │
    ▼
[2. API Key 입력]
    │ ← "Anthropic API Key를 입력해주세요"
    │ ← 플레이스홀더: "sk-ant-..."
    │ ← 하단: "API Key 발급 방법" 링크
    │
    ├──→ [Key 유효] ──→ 다음 단계
    └──→ [Key 무효] ──→ "올바른 API Key를 입력해주세요" 에러 메시지
                         (재입력 유도)
    │
    ▼
[3. 빈 오피스 진입]
    │ ← 화면: 2D 탑뷰 빈 사무실
    │ ← 중앙 안내: "나만의 AI 회사를 만들어보세요!"
    │ ← 왼쪽 팔레트: 빈 책상 아이템 (하이라이트/펄스 애니메이션)
    │ ← 툴팁: "이 책상을 드래그해서 오피스에 놓아보세요"
    │
    ▼
[4. 책상 드래그 시작]
    │ ← 책상을 클릭하면 커서에 붙음
    │ ← 오피스 그리드에 배치 가능 영역이 하이라이트
    │ ← 드래그 중 미리보기 (반투명 책상)
    │
    ▼
[5. 책상 드롭]
    │ ← 그리드에 스냅되어 배치
    │ ← 배치 완료 이펙트 (가벼운 파티클)
    │ ← 자동으로 팀 정보 입력 패널 오픈
    │
    ▼
[6. 팀 정보 입력]
    │ ← 패널 상단: "이 팀은 어떤 일을 하나요?"
    │
    │ ┌─────────────────────────────────┐
    │ │ 팀 이름: [마케팅팀          ]    │
    │ │                                 │
    │ │ 팀 설명:                        │
    │ │ [SNS 콘텐츠를 기획하고         ] │
    │ │ [작성하는 팀                   ] │
    │ │                                 │
    │ │ 예시: "고객 문의에 답변하는 팀"  │
    │ │       "블로그 글을 쓰는 팀"     │
    │ │                                 │
    │ │         [팀 만들기 →]           │
    │ └─────────────────────────────────┘
    │
    ▼
[7. AI 팀 구성 중 (로딩)]
    │ ← "팀을 구성하고 있어요..."
    │ ← 데스크 위에 로딩 애니메이션
    │ ← 2-5초 소요
    │
    ▼
[8. AI 팀 구성 제안]
    │ ← 패널에 제안된 에이전트 목록 표시
    │
    │ ┌─────────────────────────────────┐
    │ │ 🎯 마케팅팀 구성 제안            │
    │ │                                 │
    │ │ ┌─────────────────────────┐     │
    │ │ │ 📝 콘텐츠 기획 담당       │     │
    │ │ │ "주제 선정과 글 구성 담당" │     │
    │ │ └─────────────────────────┘     │
    │ │ ┌─────────────────────────┐     │
    │ │ │ ✍️ 카피라이팅 담당        │     │
    │ │ │ "실제 글 작성 담당"       │     │
    │ │ └─────────────────────────┘     │
    │ │                                 │
    │ │ [+ 에이전트 추가]               │
    │ │                                 │
    │ │   [수정하기]  [이대로 확인 ✓]   │
    │ └─────────────────────────────────┘
    │
    ├──→ [확인] ──→ 팀 생성 완료
    └──→ [수정하기] ──→ 에이전트 편집 모드
              │
              ▼
         [에이전트 삭제/수정/추가]
              │
              ▼
         [확인] ──→ 팀 생성 완료
    │
    ▼
[9. 팀 생성 완료!]
    │ ← 축하 이펙트 (파티클, 바운스 애니메이션)
    │ ← 데스크에 팀 이름 + 에이전트 아이콘 표시
    │ ← 안내: "팀이 만들어졌어요! 아래 채팅에서 일을 시켜보세요 👇"
    │ ← 채팅창에 포커스 이동
    │
    ▼
[첫 팀 생성 완료 → Flow 2로 이어짐]
```

## Flow 2: 업무 지시 → 결과 확인 (핵심 흐름)

```
┌─────────────────────────────────────────────────────────────┐
│                    FLOW 2: 일 시키고 결과 받기                │
└─────────────────────────────────────────────────────────────┘

[1. 채팅에 업무 입력]
    │ ← 채팅창 플레이스홀더: "무엇을 도와드릴까요? (예: 블로그 글 써줘)"
    │ ← 사용자 입력: "다음 주 인스타 포스팅 3개 만들어줘"
    │ ← Enter 또는 전송 버튼
    │
    ▼
[2. AI 팀 매칭]
    │ ← 채팅에 표시: "마케팅팀에게 전달할게요!"
    │ ← 0.5-1초 소요
    │
    ├──→ [매칭 성공] ──→ 다음 단계
    ├──→ [매칭 애매] ──→ "마케팅팀과 콘텐츠팀 중 어디에 맡길까요?"
    │                     (사용자 선택)
    └──→ [매칭 불가] ──→ "이 업무를 처리할 팀이 없어요. 새 팀을 만들까요?"
                          (팀 생성으로 이동 or 채팅에서 직접 처리)
    │
    ▼
[3. 오피스 시각화 시작]
    │ ← 해당 팀 데스크가 활성화 (빛남/글로우 이펙트)
    │ ← 데스크 위에 작업 진행 표시 (진행 바 또는 스피너)
    │ ← 채팅에 "마케팅팀이 작업을 시작했어요" 메시지
    │
    ▼
[4-A. 단일 팀 작업 시] ──────────────────────────────────────
    │
    │ ← 데스크 활성 상태 유지
    │ ← 클릭하면 상세 패널 오픈 (Flow 2-A)
    │
    ▼
[4-B. 복수 팀 협업 시] ──────────────────────────────────────
    │
    │ ← 첫 번째 팀(A) 데스크 활성화
    │ ← A 작업 완료 → 결과물이 팀B로 이동하는 애니메이션
    │   (빛나는 파티클/아이콘이 A에서 B로 이동)
    │ ← 이동 중 라벨: "기획안 전달 중..."
    │ ← 팀B 데스크 활성화 (A는 완료 상태로 변경)
    │ ← 이 과정 반복 (B→C→...)
    │
    ▼
[5. 작업 완료]
    │ ← 모든 관련 데스크가 "완료" 상태로 변경 (체크마크 등)
    │ ← 완료 이펙트 (가벼운 바운스/빛남)
    │ ← 채팅에 결과물 표시
    │
    │ ┌─────────────────────────────────┐
    │ │ ✅ 마케팅팀 작업 완료!            │
    │ │                                 │
    │ │ 📄 인스타 포스팅 #1              │
    │ │ "새로운 봄 컬렉션이 출시..."      │
    │ │                                 │
    │ │ 📄 인스타 포스팅 #2              │
    │ │ "주말 특가! 지금 바로..."         │
    │ │                                 │
    │ │ 📄 인스타 포스팅 #3              │
    │ │ "고객 후기 시리즈 #5..."         │
    │ │                                 │
    │ │ [결과물 자세히 보기]              │
    │ └─────────────────────────────────┘
    │
    ▼
[6. 결과 확인/활용]
    │ ← 결과물 텍스트를 복사할 수 있다
    │ ← 데스크 클릭 → 작업 히스토리에서 이전 결과도 확인 가능
    │
    ▼
[완료 → 다음 업무 지시 가능]
```

## Flow 2-A: 상세 진행 상황 확인 (서브 플로우)

```
[데스크 클릭]
    │
    ▼
[상세 패널 오픈 (오른쪽 사이드)]
    │
    │ ┌─────────────────────────────────┐
    │ │ 마케팅팀                        │
    │ │ ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━ │
    │ │                                 │
    │ │ 📋 현재 작업                     │
    │ │ "인스타 포스팅 3개 작성"          │
    │ │                                 │
    │ │ 👤 콘텐츠 기획 담당  ✅ 완료      │
    │ │   → 3개 주제 선정 완료           │
    │ │                                 │
    │ │ 👤 카피라이팅 담당  🔄 진행 중    │
    │ │   → 2/3 작성 완료               │
    │ │                                 │
    │ │ ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━ │
    │ │ 📜 이전 작업                     │
    │ │ • 블로그 글 초안 작성 (2시간 전)  │
    │ │ • 이메일 뉴스레터 (어제)          │
    │ └─────────────────────────────────┘
    │
    ▼
[패널 외부 클릭 → 패널 닫힘]
```

## Flow 3: 팀 추가 생성 (반복 흐름)

```
[오피스 화면에서]
    │
    ├──→ [팔레트에서 새 책상 드래그] ──→ Flow 1의 [4번]부터 반복
    │
    └──→ [기존 팀 데스크 옆에 배치]
         │ ← 관련 팀끼리 가까이 배치하면 시각적으로 "부서" 느낌
         │
         ▼
    [팀 정보 입력] ──→ [AI 구성] ──→ [확인] ──→ [완료]
```

## 화면 전환 요약

```
Landing ──→ API Key 입력 ──→ 빈 오피스 (온보딩)
                                   │
                              ┌────┴────┐
                              ▼         ▼
                         팀 생성     채팅 업무 지시
                         패널         │
                              │    ┌──┴──┐
                              ▼    ▼     ▼
                         오피스   팀 활성  팀 협업
                         (팀 표시) 시각화   시각화
                                   │
                                   ▼
                              상세 패널
                              (사이드)
```

**핵심 포인트**: 화면 전환은 최소화. 오피스 캔버스가 항상 메인이고, 패널과 채팅이 오버레이/사이드로 열림. 페이지 이동 없이 모든 것이 한 화면에서 이루어짐.
