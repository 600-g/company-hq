# 두근컴퍼니 외부 전경 (LoginScene) 재작성 기획

> 작성일: 2026-04-13 / 이전 시도 실패 후 전면 재시작

## 목표
로그인 화면 + EXIT 복귀 화면을 공용 사용하는 탑뷰 포켓몬 스타일 마을 씬.
HQ 건물 입구 클릭 시 사무실(OfficeScene) 진입/복귀.

## 요구사항 (두근 지시)
1. 포켓몬 에셋 기반 — 직접 그리기 금지
2. 포켓몬 폰트 사용 (Pretendard X)
3. 외부 씬 활성화 시 우측 에이전트 사이드패널 **숨김**
4. 건물 배치 자연스럽게 — 건물끼리 겹치거나 캐릭터가 건물 위로 지나가지 않게
5. 라벨("마을", "공원") 표시 금지
6. HQ 건물 명확히 구분 가능
7. 날씨/낮밤 유지 (기존 weatherCode 활용)

## 기술 결정

### 접근 방식 결정
| 옵션 | 장점 | 단점 | 결론 |
|---|---|---|---|
| A. 어나더레드 Map*.png 사용 | 완성된 맵 | 실제는 미니맵 실루엣 뿐. 전체 렌더 없음 | ✗ 불가 |
| B. .rxdata 파싱 | 원본 마을 그대로 | Ruby Marshal 바이너리. JS 파싱 불가능 | ✗ 불가 |
| C. 타일셋 조합 수동 배치 | 완전 제어, 포켓몬 룩앤필 | 타일 좌표 노동 집약 | ✅ 채택 |

## 구현 단계

### Step 1: Tilemap JSON 데이터 (수동 작성)
- `ui/public/assets/maps/company_town.json` — Tiled 포맷 or 간단 커스텀 JSON
- 30×17 레이어: ground, road, buildings, decorations

### Step 2: LoginScene 재작성
- Tilemap 로드 → 렌더
- HQ 문 영역에 invisible 클릭존
- 보행자 경로 grid로 제약
- Pokemon 폰트 적용
- showReturnBtn 분기

### Step 3: 사이드패널 숨김
- `Office.tsx`에 `outdoorActive` state
- LoginScene 활성화 시 state `true` → 사이드패널 `hidden`
- OfficeScene 복귀 시 `false`

### Step 4: 폰트 등록
- `globals.css`에 @font-face (power clear 시리즈)
- 필요시 public/에 폰트 복사 (assets 경로 접근 가능한지 확인)

### Step 5: 빌드 + 테스트
1. `npx next build` 성공
2. `deploy.sh` 배포
3. 600g.net 접속
4. 로그인 화면 확인 (마을 표시, HQ 식별 가능, 사이드패널 숨김)
5. 로그인 → 사무실 → EXIT → 같은 마을 다시 나오는지
6. HQ 문 클릭 → 사무실 복귀
7. 모바일 화면 확인

### Step 6: 이슈 수정 루프
- 스크린샷 확인 → 건물 겹침/캐릭터 건물 위 통과/사이드패널 노출 등 문제 잡기
- 수정 → 재배포 → 재확인
